┌────────────────────────────────────────────────────────────────────────────┐
│ THE PROBLEM                                                                 │
└────────────────────────────────────────────────────────────────────────────┘

  When user retakes a course:
  ✓ Old certificate gets deleted
  ✓ New certificate is issued
  ❌ New certificate has NO SIGNATURES

  Root cause: Retake logic was BLOCKING all retake certificates
  (even the first one!)

┌────────────────────────────────────────────────────────────────────────────┐
│ THE FIX                                                                     │
└────────────────────────────────────────────────────────────────────────────┘

  Changed line 83 in courseCompletionService.ts:
  
  FROM: if (enrollment && enrollment.retake_count > 0)
  TO:   if (enrollment && enrollment.retake_count > 1)
  
  Effect: Allow 1 retake certificate, block only 2nd+ retakes
