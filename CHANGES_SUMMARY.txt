================================================================================
CERTIFICATE SIGNATURE ISSUE - FIX SUMMARY
================================================================================

ISSUE:
  Certificate ID: 75b15cfa-97e0-4aff-b2ad-fdf6b14fa034
  Problem: After course retake, new certificate issued with NO signatures
  
  Timeline:
  - Day 1: User completes course → Certificate with signatures ✓
  - Day 2: User clicks "Retake" → Old cert deleted, retake_count=1
  - Day 3: User completes retake → Certificate BLOCKED, no new cert ❌

ROOT CAUSE:
  Line 83 in lib/courseCompletionService.ts had overly strict logic:
  
    if (enrollment && enrollment.retake_count > 0) {
      return { issued: false }; // Blocks ALL retakes!
    }
  
  This prevented ANY certificate after first retake.

================================================================================
SOLUTION IMPLEMENTED
================================================================================

FILE 1: lib/courseCompletionService.ts
  Location: Line 83-86
  Change: Modified retake check
  
  BEFORE:
    if (enrollment && enrollment.retake_count > 0) {
      console.log(`[CERTIFICATE_BLOCKED]...`);
      return { success: true, issued: false, ... };
    }
  
  AFTER:
    if (enrollment && enrollment.retake_count > 1) {
      console.log(`[CERTIFICATE_BLOCKED]...`);
      return { success: true, issued: false, ... };
    }
  
  Impact: Allows certificate on first retake (retake_count=1)
          Blocks only on 2nd+ retakes (retake_count≥2)

FILE 2: lib/certificateService.ts
  Location: Line 48-82
  Change: Enhanced backfill error handling
  
  Added:
  - Logging of backfill result
  - Warning if 0 signatures added
  - Final check after re-fetch
  - Better diagnostic messages
  
  Impact: Clearer error messages for troubleshooting

FILE 3: pages/api/admin/diagnose-certificate.ts (NEW)
  Purpose: API endpoint for diagnostics
  Usage: GET /api/admin/diagnose-certificate?certificateId=<id>&autoBackfill=true
  
  Returns:

  - Optional auto-repair

FILE 4: lib/certificateDiagnosticUtil.ts (NEW)

  - repairCertificate(id) - Fix one cert

  - getHealthStatistics() - System stats
